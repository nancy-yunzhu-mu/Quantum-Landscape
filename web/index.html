Turn a package's dependency tree into its real install footprint — ranked and comparable.

A live dashboard that pulls the latest release of each package from the public npm registry and converts its raw dependency list into a fast, searchable, and comparable view of each package's diversification and heaviest exposures.

Sort packages by various parameters, including concentration metrics (Largest dependency %, Top 3 dependencies %, and HHI), plus risk tags for flagging concentration risk — useful for quickly spotting hidden single-dependency exposure, bloated installs, supply-chain review, and footprint profiling.

Want another package tracked? Add it to packages.txt and hit Refresh.

Notes

Each package's direct runtime dependencies are treated as a portfolio. Every dependency is weighted by its own unpacked install size (bytes of its latest release). The concentration statistics are the same ones an analyst runs on a fund's holdings:

  • Largest % — the single heaviest dependency's share of the footprint.
  • Top 3 % — combined share of the three heaviest dependencies.
  • HHI — Herfindahl-Hirschman Index, the sum of squared shares (0–1).

The risk tag buckets the HHI using the U.S. DOJ / FTC merger-guideline thresholds (0.15 and 0.25 on the 0–1 scale): Low below 0.15, Medium below 0.25, High at or above 0.25.
